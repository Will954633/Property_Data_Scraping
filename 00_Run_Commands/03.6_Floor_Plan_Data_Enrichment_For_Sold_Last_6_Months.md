## Floor Plan Data Enrichment for `sold_last_6_months`

This is a **completely separate path** for enriching floor plans in the
`property_data.sold_last_6_months` collection.

- The **for-sale pipeline** still uses:
  - `run_production.py`
  - `analyze_single_property.py`
  - Default collection: `properties_for_sale` (from `.env` or config)

- The **sold pipeline** uses dedicated scripts that hard-code:
  - `DATABASE_NAME = "property_data"`
  - `COLLECTION_NAME = "sold_last_6_months"`

You do **not** need to touch `.env` or set `COLLECTION_NAME` for the sold pipeline.

---
## 1. Run Production for All Sold Properties with Floor Plans

```bash
cd 01.1_Floor_Plan_Data
python run_production_sold.py
```

What this does:
- Connects to `property_data.sold_last_6_months`
- Finds all documents where:
  - `floor_plans` exists and is a non-empty array, and
  - `floor_plan_analysis` does **not** exist yet
- Starts 20 workers (30s stagger) to analyze all floor plan images
- Writes results back into each document under `floor_plan_analysis`

All collection targeting for this script is controlled **inside**
`run_production_sold.py`:
- It sets `os.environ["COLLECTION_NAME"] = "sold_last_6_months"` before importing/using the batch processor

---
## 2. Test a Single Sold Property

To sanity-check one property from `sold_last_6_months` before running full production, use the dedicated single-property script:

```bash
cd 01.1_Floor_Plan_Data

# Option A: Let the script pick the first sold property with floor_plans
python analyze_single_property_sold.py

# Option B: Explicit address (example)
python analyze_single_property_sold.py "123 Example St, Robina QLD 4226"
```

This will:
1. Read from `property_data.sold_last_6_months`
2. Analyze the `floor_plans` array for that property
3. Print a detailed JSON result
4. Save a JSON file into `01.1_Floor_Plan_Data/output/`
5. Ask if you want to save the analysis back to MongoDB under `floor_plan_analysis`

The script itself sets `COLLECTION_NAME` to `sold_last_6_months` before creating the Mongo client, so it never looks at `.env` for the collection name.

---
## 3. Data Shape Expectations

Sold properties must store floor plans under:

```json
"floor_plans": [
  "https://example.com/floorplan1.jpg",
  "https://example.com/floorplan2.jpg"
]
```

The enrichment output is written to:

```json
"floor_plan_analysis": {
  // detailed structured analysis from GPT-5-nano-2025-08-07
}
```

The schema in `floor_plan_analysis` is identical for for-sale and sold properties, so downstream consumers can treat them the same way.
